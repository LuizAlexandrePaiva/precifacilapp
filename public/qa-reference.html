0. Sobre o PreciFácil

O que é

O PreciFácil é um SaaS brasileiro que ajuda freelancers, autônomos e MEIs a calcular seu preço/hora real e gerar propostas comerciais profissionais. A plataforma elimina o achismo na precificação, considerando custos fixos, horas disponíveis, impostos e margem de lucro para chegar a um valor justo e sustentável.

Para quem é

  • Freelancers de qualquer área (design, desenvolvimento, redação, marketing, etc.)
  • Profissionais autônomos que prestam serviços por projeto ou hora
  • MEIs que precisam precificar serviços de forma profissional

Qual problema resolve

A maioria dos freelancers define preços "no chute", sem considerar todos os custos envolvidos — e acaba trabalhando abaixo do valor mínimo viável. O PreciFácil transforma dados financeiros reais (custos fixos, horas produtivas, impostos, margem desejada) em três faixas de preço: Mínimo, Justo e Premium. A partir disso, o profissional pode gerar propostas comerciais padronizadas e acompanhar seu histórico de projetos.

Fluxo principal do usuário

  1. Cadastro — via email+senha ou Google OAuth. Recebe trial de 14 dias (nível Essencial).
  2. Calculadora — preenche custos fixos mensais, horas disponíveis e margem de lucro. O sistema calcula o preço/hora mínimo, justo (×1.4) e premium (×2).
  3. Proposta — com os valores calculados, cria uma proposta comercial com dados do cliente, escopo, itens inclusos/não inclusos, prazo e condições de pagamento.
  4. Exportação PDF (Pro) — gera um documento profissional com tabela de pacotes, badge "Recomendado" e validade automática.
  5. Histórico — acompanha projetos criados, valores cotados e status.
  6. Dashboard (Pro) — visualiza métricas de faturamento, propostas enviadas e taxa de aprovação.

Instruções de credenciais de teste

Não existem contas de teste pré-configuradas. O QA deve criar suas próprias contas pelo fluxo normal de cadastro (/cadastro), utilizando email real para receber os emails transacionais.

Para testar cada plano:

  • Trial (Essencial): criar conta nova — o trial de 14 dias é ativado automaticamente.
  • Grátis: usar conta com trial expirado (>14 dias) e sem assinatura ativa.
  • Essencial / Pro: assinar pelo checkout Stripe usando cartão de teste 4242 4242 4242 4242 (qualquer data futura, qualquer CVC).

1. Fluxos a Testar

Landing Page Pública

Elementos visíveis

Hero com título, subtítulo e CTA "Começar grátis" · Seção de benefícios · Tabela de planos (Grátis / Essencial R$29 / Pro R$59) · FAQ acordeão · Footer

Deve funcionar

  • CTA "Começar grátis" redireciona para /cadastro
  • Botões "Assinar" nos cards de plano redirecionam para /cadastro (se deslogado) ou checkout Stripe (se logado)
  • FAQ abre/fecha corretamente
  • Layout responsivo em mobile (≤640px) e desktop
  • Metatags SEO e OpenGraph renderizadas

Comportamento de erro

  • Se Stripe indisponível → toast "Erro ao iniciar checkout" após 10s

Cadastro /cadastro

Elementos visíveis

Campos: Nome completo, Email, Senha · Botão "Criar conta" · Separador "ou" · Botão "Continuar com Google" · Link "Já tem conta? Entrar"

Deve funcionar

  • Cadastro com email válido + senha ≥6 chars → toast "Conta criada! Verifique seu email" + redirect /login
  • Cadastro com Google OAuth → redirect para Google → retorno autenticado para /app
  • Usuário autenticado que acessa /cadastro → redirect automático para /app
  • Trial de 14 dias ativado automaticamente (verificar no banco: created_at)

Comportamento de erro

  • Email já cadastrado → toast "Este e-mail já possui uma conta. Tente Fazer login..."
  • Senha <6 chars → toast "A senha deve ter pelo menos 6 caracteres"
  • Nome vazio → toast "Informe seu nome completo"
  • Rate limit → toast "Muitas tentativas. Aguarde alguns minutos..."
  • Erro Google OAuth → toast "Erro ao entrar com Google"

Login /login

Elementos visíveis

Campos: Email, Senha · Botão "Entrar" · Botão Google · Link "Esqueceu a senha?" · Link "Criar conta"

Deve funcionar

  • Login com credenciais corretas → redirect /app
  • Login com Google → redirect /app
  • "Esqueceu a senha?" → redirect /esqueci-senha
  • Usuário já logado → redirect automático /app

Comportamento de erro

  • Email não cadastrado → toast "Não encontramos uma conta com este e-mail..."
  • Senha incorreta → mesmo toast acima (segurança — não revela se email existe)
  • Email não confirmado → toast "Confirme seu e-mail antes de entrar..."

Calculadora /app/calculadora

Elementos visíveis

Formulário: custos fixos, horas disponíveis, margem de lucro · Resultado: preço/hora mínimo, preço justo, preço premium · Botão "Criar proposta"

Deve funcionar

  • Cálculo atualiza em tempo real
  • Plano Grátis: limite de 1 cálculo/mês → após 1 uso, modal de upgrade
  • Plano Essencial/Pro: cálculos ilimitados
  • Botão "Criar proposta" leva ao formulário de proposta com dados pré-preenchidos

Comportamento de erro

  • Campos zerados ou negativos → validação impede cálculo
  • Plano Grátis tenta segundo cálculo → modal "Recurso disponível nos planos pagos"

Propostas /app/propostas

Elementos visíveis

Lista de propostas salvas · Filtros por status · Botão "Nova proposta" · Ações: visualizar, editar, exportar PDF, excluir

Deve funcionar

  • Plano Grátis: tela bloqueada → modal de upgrade
  • Plano Essencial: acesso total à lista + criar/editar propostas
  • Plano Pro: tudo do Essencial + botão "Exportar PDF" funcional
  • PDF gerado com layout correto: cabeçalho, escopo, tabela de pacotes (Mínimo/Justo/Premium), condições, footer
  • Badge "Recomendado" no Preço Justo
  • Validade calculada: data emissão + 7 dias (editável)

Comportamento de erro

  • Essencial tenta exportar PDF → modal informando que é exclusivo Pro
  • Grátis tenta acessar → tela de bloqueio com botão "Assinar agora"

Histórico /app/historico

Elementos visíveis

Tabela de projetos com: cliente, projeto, valor cotado, status, data · Filtros e ordenação

Deve funcionar

  • Plano Grátis: tela bloqueada
  • Plano Essencial/Pro: lista completa de projetos
  • Dados carregam com loading state (spinner)
  • Dados cacheados por 2 min (não refaz fetch ao voltar à página)

Comportamento de erro

  • Sem projetos → mensagem "Nenhum projeto encontrado"
  • Erro de rede → toast de erro

Dashboard /app

Elementos visíveis

Cards de métricas: Faturamento, Propostas, Taxa de Aprovação · Gráficos · Resumo do plano atual

Deve funcionar

  • Plano Grátis: tela bloqueada completamente
  • Plano Essencial: métricas de Faturamento, Propostas e Taxa de Aprovação bloqueadas; restante acessível
  • Plano Pro: acesso total a todas as métricas e gráficos
  • Dados carregam com spinner neutro (sem flash de tela de bloqueio)
  • Sem flash "Recurso disponível nos planos pagos" antes do plano carregar

Comportamento de erro

  • Plano ainda carregando → spinner centralizado (não exibe bloqueio nem conteúdo)
  • Erro ao buscar dados → toast de erro

Checkout Stripe Externo

Fluxo

Botão "Assinar agora" → loading imediato no botão → redirect para Stripe Checkout (nova aba) → pagamento → webhook atualiza plano → usuário volta ao app com plano ativo

Deve funcionar

  • Botão desabilita imediatamente ao clicar (evita clique duplo)
  • Redirect acontece em <10s
  • Plano Essencial (R$29/mês) e Pro (R$59/mês) com price IDs corretos
  • Após pagamento, webhook stripe-webhook processa evento e atualiza status
  • Email "Pagamento confirmado" enviado automaticamente

Comportamento de erro

  • Timeout >10s → toast "A conexão está demorando. Tente novamente..."
  • Erro na Edge Function → toast "Erro ao iniciar checkout"
  • Pagamento falha no Stripe → usuário permanece no plano atual

Cancelamento Portal Stripe

Fluxo

Botão "Gerenciar assinatura" → loading no botão → redirect para Stripe Customer Portal → cancelamento → webhook atualiza plano → email enviado

Deve funcionar

  • Acesso continua até o fim do período pago
  • Após expiração, reverte para plano Grátis
  • Email "Assinatura cancelada" enviado automaticamente

Comportamento de erro

  • Timeout >10s → toast de erro
  • Usuário sem assinatura ativa tenta acessar portal → erro tratado
